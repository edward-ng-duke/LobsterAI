# 04 · 编排与规模化运行（1 → 5 → 50）

目标：把单容器镜像扩展成"同机 50 个隔离实例"，每个实例有独立卷、独立访问入口、资源上限与可观测性。

---

## 1. 命名与端口规范

为可管理，给每实例一个稳定编号 `NN`（01–50）：

| 资源 | 命名 | 示例 |
|---|---|---|
| 容器名 | `lobster-NN` | `lobster-07` |
| 数据卷 | `lobster_home_NN` | `lobster_home_07` |
| noVNC 对外端口（可选，若不用反代） | `6100 + NN` | `6107` |
| 访问域名（推荐，用反代） | `inst-NN.lobster.example.com` | `inst-07...` |

> 推荐用**反向代理 + 子域名**统一入口（TLS + 认证），而不是给每实例暴露一个宿主端口。端口方案仅用于内网/调试。

---

## 2. 起步：docker-compose（适合 ≤ ~20 实例手工管理）

### 2.1 单实例服务模板
```yaml
# compose 片段：每实例复制一份，改 NN
services:
  lobster-07:
    image: lobsterai:2026.6.30
    container_name: lobster-07
    restart: unless-stopped
    shm_size: "1g"                     # 避免 Chromium /dev/shm 不足
    environment:
      - LOBSTERAI_DISABLE_GPU=1
      - SCREEN_GEOMETRY=1600x900x24
    env_file:
      - ./secrets/lobster-07.env       # 该实例的 Provider Key 等（运行时注入）
    volumes:
      - lobster_home_07:/home/lobster  # 持久化全部状态（见 03 分册）
    deploy:
      resources:
        limits:
          memory: 4g                   # 单实例内存上限（按实测调整）
          cpus: "1.5"
    networks:
      - lobsternet
    # 用反代时无需 ports；仅内网调试时开：
    # ports:
    #   - "6107:6080"

volumes:
  lobster_home_07:

networks:
  lobsternet:
```

### 2.2 用模板生成 50 份（避免手抄）
手写 50 个 service 易错。用脚本从模板渲染：

```bash
# scripts/gen-compose.sh —— 生成 docker-compose.gen.yml（含 50 个实例）
#!/usr/bin/env bash
set -euo pipefail
N=${1:-50}
IMG=${2:-lobsterai:2026.6.30}
{
  echo "services:"
  for i in $(seq -w 1 "$N"); do
    cat <<EOF
  lobster-${i}:
    image: ${IMG}
    container_name: lobster-${i}
    restart: unless-stopped
    shm_size: "1g"
    environment:
      - LOBSTERAI_DISABLE_GPU=1
    env_file:
      - ./secrets/lobster-${i}.env
    volumes:
      - lobster_home_${i}:/home/lobster
    deploy:
      resources:
        limits:
          memory: 4g
          cpus: "1.5"
    networks: [lobsternet]
    labels:
      - "traefik.enable=true"
      - "traefik.http.routers.lobster-${i}.rule=Host(\`inst-${i}.lobster.example.com\`)"
      - "traefik.http.services.lobster-${i}.loadbalancer.server.port=6080"
      - "traefik.http.routers.lobster-${i}.middlewares=lobster-auth@file"
EOF
  done
  echo "volumes:"
  for i in $(seq -w 1 "$N"); do echo "  lobster_home_${i}:"; done
  echo "networks:"
  echo "  lobsternet:"
} > docker-compose.gen.yml
echo "generated docker-compose.gen.yml with $N instances"
```

```bash
bash scripts/gen-compose.sh 50 lobsterai:2026.6.30
docker compose -f docker-compose.gen.yml up -d
```

> `secrets/lobster-NN.env` 每实例一份，放各自的 Provider Key / 代理等；**不要提交到版本库**。

## 3. 统一入口：反向代理（推荐 Traefik）

用 Traefik 按子域名把请求路由到各容器的 `6080`，并统一加 **TLS** 与 **认证**（noVNC 本身无鉴权，务必在入口挡住）。

```yaml
# 与实例同一 compose，或独立 compose
services:
  traefik:
    image: traefik:v3
    command:
      - "--providers.docker=true"
      - "--providers.docker.exposedbydefault=false"
      - "--providers.file.directory=/etc/traefik/dynamic"
      - "--entrypoints.websecure.address=:443"
      - "--certificatesresolvers.le.acme.email=you@example.com"
      - "--certificatesresolvers.le.acme.storage=/acme/acme.json"
      - "--certificatesresolvers.le.acme.tlschallenge=true"
    ports:
      - "443:443"
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock:ro
      - ./traefik/dynamic:/etc/traefik/dynamic:ro
      - ./traefik/acme:/acme
    networks: [lobsternet]
```

`traefik/dynamic/auth.yml`（Basic Auth 中间件示例，生产建议接 OAuth/SSO 转发认证）：
```yaml
http:
  middlewares:
    lobster-auth:
      basicAuth:
        users:
          - "user07:$apr1$....hashed...."   # htpasswd 生成，每实例可用不同账号
```

> WebSocket：noVNC 走 WS，Traefik/Caddy/nginx 默认支持 WS 升级；用 nginx 时记得 `proxy_set_header Upgrade/Connection`。

**访问方式**：用户打开 `https://inst-07.lobster.example.com/vnc.html`，过认证后即见该实例的 LobsterAI 桌面。

## 4. 资源限制与稳定性（同机 50 实例的关键）

- **务必设每容器 `memory` 上限**：防止单个实例内存膨胀拖垮整机（触发宿主 OOM 杀错进程）。按 07 分册实测值设（初值 4g，观测后收敛到 2–3g）。
- **设 `cpus` 软配额**：避免 50 个并发冷启动/推理时 CPU 争抢导致雪崩。
- **错峰启动**：50 个容器同时起会有 CPU/IO 风暴（每个都要拉起 Electron + OpenClaw 网关，冷启 2–12s）。用脚本分批 `up -d`（如每批 5 个、间隔 20s）。
- **`shm_size: 1g`**：Chromium 需要；代码已 `disable-dev-shm-usage`，但给足更稳。
- **重启策略 `unless-stopped`**：宿主重启后自动拉起；配合 supervisord 内部各进程自愈。
- **日志上限**：给 Docker 配 `json-file` 的 `max-size/max-file`，否则 50 实例日志会撑爆磁盘。
  ```json
  // /etc/docker/daemon.json
  { "log-driver": "json-file", "log-opts": { "max-size": "20m", "max-file": "3" } }
  ```

## 5. 可观测性

| 维度 | 手段 |
|---|---|
| 容器存活/重启 | `docker events`、cAdvisor + Prometheus |
| 内存/CPU/网络 | cAdvisor → Prometheus → Grafana，每实例一行 |
| 卷用量 | 定时 `du -sh` 各卷或云盘监控，设阈值告警 |
| 应用健康 | 容器 `HEALTHCHECK`（noVNC 端口）；进阶：探 OpenClaw 网关 `/health`（需 token，见 `openclawEngineManager.ts`） |
| noVNC 可达 | 反代侧健康检查 / 合成拨测 |
| 应用日志 | 各实例 `~/.config/LobsterAI/logs/`（主进程）与 supervisord 各程序日志；可挂日志卷或用 Promtail 收集 |

## 6. 规模化后的选择：从 compose 到编排器

50 个同机实例 compose 尚可管理，但若要**多机、弹性、滚动升级、故障自愈更强**，迁移到编排器：

- **Kubernetes**：每实例一个 `StatefulSet`（`replicas: 1`）+ 独立 `PVC`（保证卷与身份稳定绑定），或用一个 `StatefulSet(replicas:50)` + `volumeClaimTemplates`（每 Pod 一 PVC）。`Service` + `Ingress`(nginx/Traefik) 做子域名路由与认证。资源用 `requests/limits`。
  - 关键点：StatefulSet 的稳定网络标识与 PVC 绑定天然契合"一实例一状态一入口"。
- **Nomad / Docker Swarm**：更轻量，适合单/少数机器。
- **裸机多机 + compose**：把 50 拆到 N 台机器，每台跑一批，反代统一入口。

> 分片建议：单机内存是硬约束（07 分册）。若单机放不下 50，按内存把实例分摊到多台，反代层对用户透明。

## 7. 生命周期常用操作

```bash
# 扩容：改 gen 脚本的 N，重新生成并 up
bash scripts/gen-compose.sh 60 && docker compose -f docker-compose.gen.yml up -d

# 升级镜像（逐个滚动，保留卷）
docker compose -f docker-compose.gen.yml pull
docker compose -f docker-compose.gen.yml up -d --no-deps lobster-07   # 单个滚动

# 优雅停某实例（触发 SQLite 刷盘，见 03 分册 stopwaitsecs）
docker stop --time 30 lobster-07

# 销毁实例但保留数据（仅删容器，不删卷）
docker rm -f lobster-07     # 卷 lobster_home_07 仍在，可随时重建
```
