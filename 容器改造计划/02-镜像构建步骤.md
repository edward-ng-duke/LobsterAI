# 02 · 镜像构建步骤（Linux + Xvfb + noVNC）

目标：产出一个镜像，容器启动后在虚拟显示上运行 LobsterAI，并通过 noVNC 暴露到浏览器。采用**两阶段构建**：`builder` 阶段编译打包，`runtime` 阶段只装运行所需系统库 + 虚拟显示/远程桌面。

> 约定：以下命令中 `linux-x64` 为例；ARM 服务器请把所有 `linux-x64` 换成 `linux-arm64`，基础镜像换成对应架构。原生模块**不可跨架构复用**。

---

## 0. 前置确认（构建前先跑一遍）

1. **构建需要联网**：`predist:linux` 会执行 `openclaw:runtime:linux-x64`，其中 `openclaw:ensure` 会 `git clone github.com/openclaw/openclaw` 指定 tag（`package.json` 里 `openclaw.version = v2026.6.1`），`openclaw:plugins` 会从 npm / 自定义 registry / git 安装插件。构建机/构建阶段必须能访问 GitHub、npm registry 以及 `https://npm.nie.netease.com`（POPO 插件，`optional`，可跳过）。
   - 依据：`scripts/ensure-openclaw-version.cjs`、`scripts/ensure-openclaw-plugins.cjs`、`package.json:10-67`。
2. **产物路径与可执行名（已从配置确认）**：`electron-builder.json` 中 `directories.output = "release"`、`productName = "LobsterAI"`、`executableName = "LobsterAI"`。因此产物在 `release/linux-unpacked/`，可执行文件名为 **`LobsterAI`**（大写驼峰），完整路径 `release/linux-unpacked/LobsterAI`。打包后仍建议 `ls` 复核一次。
3. **Node 版本**：仓库要求 `>=24.15 <25`，构建基础镜像用 `node:24-bookworm`。
4. **原生模块**：`npm ci` 的 `postinstall` 会跑 `patch-package && electron-builder install-app-deps`，把 `better-sqlite3` / `sharp` / `node-nim` 等按 Electron ABI 重新编译。需要 `python3` + `build-essential` + `git`。

---

## 1. 关键构建命令（在仓库根，本机先验证一次）

先在一台 Linux 构建机上手动跑通，再固化到 Dockerfile：

```bash
# 1) 安装依赖（触发原生模块按 Electron 重建）
npm ci

# 2) 一步到位：构建 Linux 版 OpenClaw runtime + 渲染层 + 主进程 + 技能 + 打包
#    predist:linux 会自动先构建 linux-x64 runtime
npm run dist:linux

# 3) 确认产物
ls -la release/linux-unpacked/          # 可执行文件在此
ls -la release/linux-unpacked/resources # 应含 cfmind（OpenClaw runtime）
```

> 若只想快速验证"能不能跑起来"，也可以不打包，直接在开发模式下起（`npm run electron:dev:openclaw`），但**生产镜像请用打包产物**，更小更稳。

## 2. Dockerfile（两阶段）

将下面文件放到仓库根目录 `Dockerfile`（或 `deploy/Dockerfile`）。

```dockerfile
# ============ Stage 1: builder ============
FROM node:24-bookworm AS builder

# 原生模块与 OpenClaw runtime 构建所需
RUN apt-get update && apt-get install -y --no-install-recommends \
      git python3 build-essential ca-certificates bash \
    && rm -rf /var/lib/apt/lists/*

WORKDIR /app
COPY package*.json ./
# 允许 postinstall 的 patch-package / install-app-deps 找到补丁与配置
COPY patches ./patches
COPY scripts ./scripts
RUN npm ci

# 拷贝其余源码后打包
COPY . .

# 构建：predist:linux 会先构建 linux-x64 的 OpenClaw runtime（需要联网）
#   如需走代理：ENV HTTP_PROXY/HTTPS_PROXY=... （见下方"网络与代理"）
RUN npm run dist:linux \
    && ls -la release/linux-unpacked

# ============ Stage 2: runtime ============
FROM debian:bookworm-slim AS runtime

ENV DEBIAN_FRONTEND=noninteractive
# Electron/Chromium 运行所需系统库 + 虚拟显示 + 远程桌面 + 进程管理
RUN apt-get update && apt-get install -y --no-install-recommends \
      # Electron/Chromium 运行依赖
      libgtk-3-0 libnotify4 libnss3 libxss1 libxtst6 xdg-utils \
      libatspi2.0-0 libdrm2 libgbm1 libxcb-dri3-0 libasound2 \
      libx11-xcb1 libxcomposite1 libxdamage1 libxrandr2 libxfixes3 \
      libxkbcommon0 libpango-1.0-0 libcairo2 libcups2 libatk1.0-0 \
      libatk-bridge2.0-0 \
      # 虚拟显示 + 远程桌面
      xvfb x11vnc novnc websockify openbox \
      # 进程管理与信号
      supervisor tini \
      # 运行期可能用到（fonts-noto-cjk 避免乱码）
      python3 git ca-certificates fonts-noto-cjk \
      # ⚠️ 浏览器：登录用 shell.openExternal 打开外部浏览器做 OAuth，容器内必须有一个
      chromium \
    && rm -rf /var/lib/apt/lists/*

# 非 root 运行
RUN useradd -m -s /bin/bash -u 10001 lobster
ENV HOME=/home/lobster \
    DISPLAY=:99 \
    LOBSTERAI_DISABLE_GPU=1 \
    BROWSER=chromium \
    NODE_ENV=production \
    NOVNC_PORT=6080 \
    VNC_PORT=5900 \
    SCREEN_GEOMETRY=1600x900x24

# 拷贝打包产物（可执行文件为 /opt/lobsterai/LobsterAI）
COPY --from=builder /app/release/linux-unpacked /opt/lobsterai

# 进程编排与启动脚本
COPY deploy/supervisord.conf /etc/supervisor/conf.d/lobsterai.conf
COPY deploy/entrypoint.sh    /usr/local/bin/entrypoint.sh
COPY deploy/start-app.sh     /usr/local/bin/start-app.sh
RUN chmod +x /usr/local/bin/entrypoint.sh /usr/local/bin/start-app.sh \
    # chrome-sandbox 在 no-sandbox 模式下仍可能被检查，修正权限或依赖 --no-sandbox
    && (chmod 4755 /opt/lobsterai/chrome-sandbox || true) \
    # 把 chromium 设为系统默认浏览器，供登录 OAuth 的 xdg-open/openExternal 调用
    && (update-alternatives --set x-www-browser /usr/bin/chromium || true) \
    && mkdir -p /home/lobster/.config /home/lobster/projects \
    && chown -R lobster:lobster /home/lobster /opt/lobsterai

USER lobster
EXPOSE 6080

# tini 作 PID1 转发信号；supervisord 编排各进程
ENTRYPOINT ["/usr/bin/tini", "-g", "--"]
CMD ["/usr/local/bin/entrypoint.sh"]

# 健康检查：noVNC 端口可达即认为存活（更严格可探 OpenClaw 网关健康）
# 用 bash exec 形式，因 /dev/tcp 是 bash 特性，默认 sh(dash) 不支持
HEALTHCHECK --interval=30s --timeout=5s --start-period=90s --retries=3 \
  CMD ["bash","-c","exec 3<>/dev/tcp/127.0.0.1/${NOVNC_PORT}"]
```

> 包名基于 Debian **bookworm**。换基础镜像版本（如 trixie）时需复核：`libasound2` 在 trixie 改名为 `libasound2t64`，个别 `lib*` 也可能带 `t64` 后缀。正式化前用 `apt-cache show <pkg>` 逐个核对可安装性。已刻意剔除在 bookworm 中不存在的 `libgconf-2-4`。

## 3. 启动脚本

### `deploy/entrypoint.sh`
```bash
#!/usr/bin/env bash
set -euo pipefail

# 清理上次残留的 X 锁（容器重启后 Xvfb 可能留下 /tmp/.X99-lock）
rm -f /tmp/.X99-lock /tmp/.X11-unix/X99 2>/dev/null || true

exec /usr/bin/supervisord -c /etc/supervisor/conf.d/lobsterai.conf
```

### `deploy/start-app.sh`
```bash
#!/usr/bin/env bash
set -euo pipefail

# 等待 Xvfb 就绪
for i in $(seq 1 30); do
  if xdpyinfo -display "${DISPLAY}" >/dev/null 2>&1; then break; fi
  sleep 0.5
done

# 启动 LobsterAI（executableName=LobsterAI，见 electron-builder.json）
# --no-sandbox 与代码内 no-sandbox 双保险；HOME 已指向持久卷
exec /opt/lobsterai/LobsterAI --no-sandbox --disable-gpu
```

### `deploy/supervisord.conf`
```ini
[supervisord]
nodaemon=true
user=lobster
logfile=/home/lobster/supervisord.log
pidfile=/home/lobster/supervisord.pid

# 1) 虚拟显示
[program:xvfb]
command=/usr/bin/Xvfb :99 -screen 0 %(ENV_SCREEN_GEOMETRY)s -nolisten tcp -ac
autorestart=true
priority=10
stdout_logfile=/home/lobster/xvfb.log
stderr_logfile=/home/lobster/xvfb.err.log

# 2) 轻量窗口管理器（可选，改善窗口最大化/焦点）
[program:openbox]
command=/usr/bin/openbox
autorestart=true
priority=20
environment=DISPLAY=":99"
stdout_logfile=/home/lobster/openbox.log
stderr_logfile=/home/lobster/openbox.err.log

# 3) x11vnc：把 :99 暴露为 VNC（生产建议设 -passwd 或用 -rfbauth）
[program:x11vnc]
command=/usr/bin/x11vnc -display :99 -forever -shared -rfbport %(ENV_VNC_PORT)s -nopw
autorestart=true
priority=30
stdout_logfile=/home/lobster/x11vnc.log
stderr_logfile=/home/lobster/x11vnc.err.log

# 4) noVNC：VNC → WebSocket，浏览器访问 http://<host>:6080/vnc.html
[program:novnc]
command=/usr/bin/websockify --web /usr/share/novnc %(ENV_NOVNC_PORT)s localhost:%(ENV_VNC_PORT)s
autorestart=true
priority=40
stdout_logfile=/home/lobster/novnc.log
stderr_logfile=/home/lobster/novnc.err.log

# 5) LobsterAI 本体（最后启动，给足退出时间以刷盘 SQLite）
[program:lobsterai]
command=/usr/local/bin/start-app.sh
autorestart=true
priority=50
stopsignal=TERM
stopwaitsecs=25
environment=DISPLAY=":99",HOME="/home/lobster",LOBSTERAI_DISABLE_GPU="1"
stdout_logfile=/home/lobster/lobsterai.log
stderr_logfile=/home/lobster/lobsterai.err.log
```

## 4. 构建与本地冒烟测试

```bash
# 构建镜像（可能耗时较久：需拉取 OpenClaw 源与插件）
docker build -t lobsterai:2026.6.30 -f Dockerfile .

# 单容器冒烟：映射 noVNC，挂一个持久卷到 HOME，给足 shm
docker run -d --name lobster-smoke \
  --shm-size=1g \
  -p 6080:6080 \
  -v lobster_home_smoke:/home/lobster \
  lobsterai:2026.6.30

# 浏览器打开 http://<服务器IP>:6080/vnc.html ，应看到 LobsterAI 界面
docker logs -f lobster-smoke        # 观察各进程与 OpenClaw 网关启动日志
```

**冒烟要点：**
- 界面能加载、能登录、能新建会话跑一次对话（验证 OpenClaw 网关起来了）。
- 添加一个 Skill、上传一个文件，`docker restart lobster-smoke` 后仍在（验证持久化，详见 03 分册）。
- `docker stats lobster-smoke` 观察内存峰值，为容量规划取数（详见 07 分册）。

## 5. 网络与代理（构建阶段）

若构建机在内网、需代理访问 GitHub/npm：

```dockerfile
# builder 阶段顶部加入（按需）
ENV HTTP_PROXY=http://proxy:port \
    HTTPS_PROXY=http://proxy:port \
    NO_PROXY=localhost,127.0.0.1
# npm registry 也可指定： RUN npm config set registry https://registry.npmmirror.com
```

- POPO 插件（`moltbot-popo`，registry `npm.nie.netease.com`）标记 `optional`，外网构建可跳过或移除，不影响主流程。
- NIM 渠道插件来自 `git+github`，同样 `optional`。

**离线/可复现构建建议**：把 `openclaw:runtime:linux-x64` 的产物（`vendor/openclaw-runtime/`）与插件预先构建好、纳入版本库或制品库，构建时用 `OPENCLAW_SKIP_ENSURE=1` 跳过联网克隆，显著提升可复现性与速度。

## 6. 常见坑速查

| 现象 | 原因 | 处理 |
|---|---|---|
| 容器起来但 noVNC 黑屏 | Xvfb 未就绪 / 应用崩溃 | 看 `lobsterai.err.log`、`xvfb.err.log`；确认可执行名正确 |
| `chrome-sandbox` 报错退出 | SUID 权限/沙箱 | 已 `--no-sandbox`；必要时 `chmod 4755 chrome-sandbox` 或用 `--cap-add=SYS_ADMIN`（不推荐） |
| 渲染卡顿/CPU 高 | 软件渲染 | 正常（无 GPU）；`LOBSTERAI_DISABLE_GPU=1` 已启用软件光栅 |
| `/dev/shm` 不足致渲染崩溃 | 默认 shm 64MB | `--shm-size=1g` 或 `--tmpfs /dev/shm`；代码已设 `disable-dev-shm-usage` |
| 原生模块 `invalid ELF` | 跨架构复用 | 必须在目标架构上构建 runtime 与原生模块 |
| 界面中文乱码 | 缺 CJK 字体 | 安装 `fonts-noto-cjk` |
| OpenClaw 网关起不来 | runtime 未打进镜像 | 确认 `resources/cfmind` 存在于 `linux-unpacked` |
| **点击登录无反应/报错，无法进入** | 容器内无浏览器，`shell.openExternal`→`xdg-open` 失败（`main.ts:4588`） | **镜像内装 `chromium` 并设默认**（本文已含）；`BROWSER=chromium`；仍不通则考虑预置 token 或改造登录（见 [05 分册](05-代码与配置改造点.md) 与 [08 · A3](08-实施路线图与验收清单.md)） |

打通单容器后，进入 [03-状态持久化与卷映射.md](03-状态持久化与卷映射.md)。
