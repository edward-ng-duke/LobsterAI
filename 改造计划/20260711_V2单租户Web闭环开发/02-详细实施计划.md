# V2 详细实施计划

## 1. 不可缩小的目标

最终交付必须证明浏览器内部 Alpha 能在生产同构、完全断公网环境完成：OIDC 登录 → 创建会话并提交首 prompt → OpenClaw 真实 turn → REST/WS 流式响应 → 基础 Artifact 查看。范围包含 PR-0～PR-4 和 V1 全部前置硬门；不以 mock、noVNC、Electron UI 或本机 SQLite 替代真实验收。

## 4. 工程边界

- 目标应用：`apps/web`、`apps/api`、`apps/worker`、`apps/runtime-orchestrator`。
- 共享事实源：`libs/shared/contracts`；纯类型 `libs/shared/types`；Web bridge `libs/client/bridge`；DB/auth `libs/server/*`。
- 数据与部署：`prisma/`、`docker/`、`charts/lobsterai/`；旧 `src/` 保持桌面兼容，不把 SaaS 业务继续堆入 `src/main/main.ts`。
- `coworkStore.ts` 和 `openclawConfigSync.ts` 是高风险重写/移植，不得整文件搬运，也不得顺手做无关拆分。
- OpenClaw 仍是唯一 runtime；先找 LobsterAI 集成点，确无钩子时才用版本化 patch。

## 5. 生产网络基线（覆盖旧计划冲突）

根 AGENTS 的“生产机器不得访问公网”高于旧计划中的“egress-proxy 唯一外网出口”。本交付采用以下可执行口径：

- 生产 namespace、节点和 Pod 对公网默认拒绝，且无公网 NAT/代理兜底；`egress-proxy` 若保留，只作为内网审计/路由服务，禁止转发公网。
- 模型/ASR、OIDC、S3/MinIO、Postgres、Redis、registry、npm/插件/skill 包源均必须是批准的内网地址；供应商能力由内部镜像、内部模型网关或批准的离线同步流程提供。
- 测试/镜像构建环境可联网下载并固化依赖，但成品以 digest/lockfile/SBOM 进入内网；生产不得运行 `npm install`、`pip install` 或动态公网抓取。
- V103 必须同时证明批准内网服务可达和任意公网/metadata/RFC1918 非批准目标不可达。无批准内网模型或 OIDC 时，真实 V2 验收为 BLOCKED，不能临时放公网。

## 6. 交付与回滚总策略

- 每批次使用独立 feature flag/Helm values；数据库先向前兼容迁移，再发布读写，回滚应用不得反向破坏数据。
- Web 与 Electron 双跑；关闭 Web 路由/Ingress 可回滚，不删除桌面路径。
- contracts 与生成物同源回滚；禁止只回滚生成文件。
- runtime 镜像、chart、migration、OpenClaw patch 均固定版本/digest；阶段门证据绑定 SHA。
- 任一硬门失败：停止依赖任务，回滚本任务变更并在风险表登记，不以 waiver 勾总体表。
