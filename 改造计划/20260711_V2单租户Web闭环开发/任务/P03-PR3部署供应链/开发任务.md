## 1. 目标、非目标与事实源

- 目标：把 P00 的 `docker/*`、`charts/lobsterai`、`docs/supply-chain` 占位物变成可重复构建、可 lint、可审计、默认安全的 PR-3 交付物，并将三个 gate 从 `NOT_APPLICABLE`/exit 78 激活为真实阻断门。
- Docker 目标边界：`web`、`api`、`worker`、`runtime-orchestrator`、`openclaw-runtime` 五个生产镜像；debug 镜像使用独立 Dockerfile/tag/profile，禁止被生产 values 引用。
- Helm 目标边界：Web/API/worker/runtime-orchestrator 的常驻工作负载及 Service、配置、安全/资源占位；sandbox 仅保留可 lint 的受限 Pod/RuntimeClass 参数入口。
- 供应链目标边界：仓库内 bundled skills、OpenClaw plugins、Kit/MCP 默认资产的确定性 inventory，及构建镜像的 digest、SBOM、签名/漏洞策略。
- 权威冲突顺序：根 `AGENTS.md` 的生产断公网政策 > `附录C` 决策 > `19` PR-3 边界 > `07`/`10`/`14`/`15`/`16`；生产节点和容器不得直接访问公网。
- 非目标：不交付 Ingress 公网域名/TLS 实例、不提交真实 Secret/tenant 数据、不发布外部 registry、不证明 gVisor/Kata 性能、不实现业务 API/队列/编排逻辑。

## 3. 实施序列（Developer 必须 Red → Green）

1. 先建立隔离副本测试工具与最小恶意 fixtures；逐项证明当前 P00 占位物会因目标行为缺失而 Red。
2. 先实现供应链 inventory/schema/checker 纯逻辑及 mutation 用例，再激活 `supply-chain:check`；避免先改 stage manifest 造成无诊断失败。
3. 实现五个生产 Dockerfile 和 checker；builder 可在批准的构建环境取依赖，runtime stage 必须只复制锁定产物且在断网下启动。
4. `openclaw-runtime` 使用 `package.json.openclaw` 的 pinned 版本/插件清单和 Linux 目标构建产物；不得复制宿主 macOS runtime、Electron UI、Xvfb/noVNC、`.npmrc`、缓存、源码 Secret。
5. 实现 Helm values/schema/templates 与负向渲染测试；先用固定假 digest/template-only 验证，不连接真实集群。
6. 将三个 gate 更新为 `ACTIVE`/exit 0 only-on-pass；runner 调真实 checker并生成新鲜、原子写、绑定 SHA/invocation 的报告，checker 失败原样阻断。
7. CI 对三个 active gate 直接执行真实 npm 命令；仍 deferred 的 gates 继续通过 verifier 期待 exit 78，禁止把 ACTIVE gate 交给旧 `expect-*` 假装通过。
8. Refactor 只抽取稳定的路径、schema 和进程执行 helper；不夹带业务功能、V1 PoC 或 V4 orchestration。

## 4. 验收标准（15 项，缺一不可）

- `P03-AC01`：五个生产 Dockerfile 都执行真实多阶段 build；输入绑定 lockfile/pinned runtime，重复构建不从未声明上下文取文件，任一 Dockerfile/目标缺失即 gate 失败。
- `P03-AC02`：五个最终镜像均为数字 UID/GID 非 root，`USER` 生效，禁止特权/capability 依赖；根文件系统按运行需要最小化，只有声明的数据/临时目录可写。
- `P03-AC03`：Web/API/worker/orchestrator/runtime 分别有有效启动或健康/就绪探针；worker/orchestrator 接收 SIGTERM 后在限定时间退出，僵尸/吞信号/无限重启测试失败。
- `P03-AC04`：生产 runtime stage 在 `--network=none` 或等价断公网 smoke 中不执行 `npm install/npx/pip/curl/git clone`；所有 Node/Python/native 依赖已在批准 build stage 固化，缺依赖必须 fail closed。
- `P03-AC05`：`openclaw-runtime` 镜像能执行版本/启动前置 smoke，版本等于 pinned OpenClaw；内容扫描证明不含 Electron、Chromium GUI、Xvfb/x11vnc/noVNC、真实 Secret、tenant 数据或 macOS binary。
- `P03-AC06`：生产与 debug 构建入口、repository/tag、values/profile 完全分离；生产 chart 对 `debug`、`latest`、无 digest、noVNC/Xvfb 字样拒绝渲染。
- `P03-AC07`：chart 提供 `values.schema.json` 并约束 required、类型、资源值、repository+`sha256:<64hex>` digest、RuntimeClass 占位；未知关键字段或空生产值无法 lint/render。
- `P03-AC08`：四个常驻组件渲染 Deployment，Web/API 的必要 Service 正确选中 Pod；ConfigMap 只含非敏感配置，Secret 只按 name/key 引用，渲染输出无真实 Secret 值。
- `P03-AC09`：Pod securityContext 默认 `runAsNonRoot`、`allowPrivilegeEscalation:false`、drop `ALL`、seccomp RuntimeDefault、只读根文件系统、禁自动挂 SA token；resources requests/limits 和探针不可被生产 values 静默清空。
- `P03-AC10`：chart 渲染 default-deny ingress/egress NetworkPolicy 与最小 DNS/明确内网 service 规则、namespace ResourceQuota；不存在 `0.0.0.0/0` 公网 egress、IMDS/RFC1918 泛放行。
- `P03-AC11`：runtime-orchestrator 使用独立 ServiceAccount 和最小 namespaced RBAC 占位；其他组件不得获得 K8s write，sandbox 默认 disabled，RuntimeClass 与 sandbox SA 缺失时启用即 hard fail。
- `P03-AC12`：inventory schema/manifest 双向覆盖现有 bundled skills、OpenClaw plugins、Kit/MCP defaults；重复 ID、大小写路径碰撞、symlink 越界、缺失/额外资产、可选插件状态、unpinned/latest、integrity 漂移均阻断。
- `P03-AC13`：每个镜像生成机器可读 SBOM并关联 image digest/source SHA；策略拒绝缺 SBOM、digest 不匹配、Critical 漏洞/恶意 skill finding，例外必须有 owner、reason、expiresAt 且过期自动失败。
- `P03-AC14`：发布/部署策略只接受 digest 与受信签名身份；内部 registry/信任根未冻结时 publish/verify 明确 `BLOCKED`，但本地 build+SBOM gate 不伪造签名或上传公网；Secret 不出现在 image history/SBOM/report。
- `P03-AC15`：`supply-chain:check`、`docker:build:check`、`helm:lint` 从 P00 exit 78 变为真实检查，成功才 exit 0；篡改 runner/checker/hash/report、删除 fixture、工具缺失、检查失败均非零且 CI 阻断。
