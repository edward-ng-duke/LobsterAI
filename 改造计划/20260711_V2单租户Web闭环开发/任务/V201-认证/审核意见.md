# 审核意见

> Planner 仅预置独立 Reviewer 的审核协议；当前没有审核结论。Reviewer 不得复用 Planner/Developer Agent，不得替 Developer 补大块实现或把 BLOCKED 写成 APPROVED_FOR_TEST。

## Reviewer 启动门

- Developer 已交 REVIEW_PENDING，稳定 SHA、Red commits、20 AC 对账和证据目录齐全。
- V107=go、P01/P02协调者ACCEPTED、OIDC-D0-01～14和真实内网IdP均可由原始证据验证。
- Reviewer 从干净工作树读取完整计划、Developer记录、diff、contracts/generated、migration、配置、fixture与真实IdP报告。
- 任一门缺失：写 CHANGES_REQUESTED 或 BLOCKED，禁止交Tester。

## 必审清单

| 领域 | 必须独立确认 |
|---|---|
| 硬门/三态 | V107 go与P02 accepted不可伪造；PASS=0、FAIL=1、BLOCKED=2；skip/fixture不解除真实IdP门 |
| 契约 | Auth route/DTO/error/security均来自libs/shared/contracts；生成物未手改；nonce/client/redirect/cookie/ticket语义与实现一致 |
| OIDC/PKCE | 两层state/nonce/transaction单用；S256；code/client/redirect/challenge绑定；callback mix-up防护 |
| JWT/JWKS | 固定alg、kid、iss、aud/azp、sub、tenant claim、iat/nbf/exp/max-age；未知kid有界刷新和single-flight；旧新key重叠 |
| identity/session | iss+sub关联；active user/membership；内部tid来自DB；code/refresh只存hash；cookie/CSRF/CORS与header预算 |
| rotation/revocation | family锁/CAS、grace合法并发、超窗reuse撤销、absolute expiry、logout/disabled/back-channel与sid/ticket联动 |
| API/WS | REST与WS重建同一AuthContext；ticket短期单用绑定scope/sid/jti；禁URL JWT/Cookie/客户端tid |
| 数据/迁移 | 只新增forward migration；RLS ENABLE+FORCE、唯一索引、原子消费与非特权真实PG证据；不改P02历史 |
| 断公网/秘密 | 仅批准内网IdP；无runtime下载/公网fallback；secret只为secref，token/code/cookie不进Git/DB明文/URL/log/trace |
| 双证据/回滚 | 可控fixture覆盖攻击面且真实IdP覆盖happy/negative/rotation；feature flag、key、应用与forward-fix回滚可执行 |

Reviewer 必须亲自运行至少十二个、覆盖以下全部类别的隔离 mutation，不能只引用 Developer 输出：

1. 将 S256 改为 plain 或跳过 verifier。
2. 跳过 SPA state、upstream state 或 nonce 任一校验。
3. 用另一 issuer/client 的 callback/code 做 mix-up。
4. 接受 alg=none、HS*、未知alg、伪kid或token自带jku/x5u。
5. 改错 iss、aud、azp、空sub、tenant claim或内部tid。
6. 未知kid并发触发JWKS请求风暴；旧key在重叠窗口内/外边界。
7. wildcard、scheme-relative、userinfo、fragment、双编码redirect与恶意return_to。
8. code过期、重放与两个并发exchange。
9. 同family至少20并发refresh、迟到响应、grace内与超grace重放。
10. 缺CSRF/恶意Origin/CORS *+credentials/错误Cookie flag。
11. 过大JWT、重复Authorization、超大Cookie或总header。
12. ticket复用/过期/撤销后消费/REST与WS身份字段不一致/URL JWT。
13. logout与refresh竞态；IdP end-session不可用时本地撤销。
14. 若支持back-channel：伪签名、错events/aud、缺sid/sub、jti重放；不支持则核对明确N/A。
15. 把issuer/JWKS/token endpoint换成公网或未批准RFC1918地址。
16. 把client secret、私钥、refresh或code canary写入日志/DB/Git，扫描必须失败。
