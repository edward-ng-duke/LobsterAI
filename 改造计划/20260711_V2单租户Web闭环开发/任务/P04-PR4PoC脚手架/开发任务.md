## 1. 目标、范围与非目标

### 1.1 目标

将 P00 的 `poc:v1:check` 占位入口、PoC manifest、证据模板，以及 P03 验收后的 runtime 镜像和 Helm sandbox 样例，收敛为可重放、fail-closed、可机读的 V1 PoC harness。它为 V101～V106 提供 `acquire / real-turn / egress / storage`、gVisor 兼容/性能矩阵和 k6 流式客户端执行入口，并生成绑定 source SHA、runId、真实集群和镜像 digest 的证据包。

P04 交付的是“**可执行的测量仪器与证据契约**”，不是 V1 本身。`poc:v1:check=PASS` 只表示 harness 完整且 Day-0 preflight 真实可用，不表示 V1.0～V1.6 已通过。

### 1.2 范围内

- `apps/runtime-orchestrator/poc/`：runner、preflight、四类 probe 接口、k6/WS 最小客户端、schema、fixtures、cleanup 和 README。
- `charts/lobsterai/templates/sandbox-pod.yaml` 及必要的 P04 专用 values/render 入口：只让样例可参数化和 server-side dry-run，不做生产编排器。
- `docs/poc/YYYYMMDD_V1_OpenClaw沙箱PoC证据包/`：证据契约与空模板；实际 run 必须进入带日期、主题和唯一 runId 的目录，禁止 `outputs/`。
- `scripts/check-v1-poc.mjs`（或等价入口）、`scripts/saas-stage-gates.json`、`package.json`、`.github/workflows/saas-scaffold.yml`：激活真实 P04 gate，并延续 stage runner 的 SHA、新鲜度与原子报告语义。
- `tests/poc-v1-*.test.ts`、`tests/fixtures/poc_v1/`：纯逻辑、隔离副本 mutation 和恶意 manifest/evidence/namespace 用例。

### 1.3 非范围

- 不声称或勾选 V1.0～V1.6；fixture server/mock kubectl 不能替代 V101～V106 的真实集群证据。
- 不实现 V4 生产 Runtime Orchestrator、动态 namespace/PVC/Pod 生命周期、预热池、租约协调、自愈或多副本路由。
- 不改 OpenClaw 业务逻辑，不重写 Config Sync/Cowork/File Service/Web bridge，不新增第二 runtime。
- 不安装或迁移 Playwright 产品 E2E；k6/WS PoC client 不等于 `test:e2e`。若 P00 仍把其 `activationTask` 写为 P04，应改正 deferred owner/reason，不能冒充浏览器绿灯。
- 不提交 credential、kubeconfig、真实租户数据、大日志/镜像 tar，不发公网 registry，不让生产 runtime 下载依赖。

## 2. 依赖硬门与状态语义

### 2.1 P03 不可豁免硬门

Developer 开始前必须在开发记录附 P03 协调者验收、Tester `PASSED`、稳定 SHA，以及下列新鲜重放：

```bash
npm run supply-chain:check
npm run docker:build:check
npm run helm:lint
```

任一缺失，P04 保持 `IMPLEMENTATION_BLOCKED_BY_P03`。禁止“先实现再等 P03”。

### 2.2 Day-0 与三态结果

- `PASS`：检查实际执行且断言全过，有同 runId 新鲜证据；仅此状态返回 0。
- `FAILED`：输入合法但目标行为/安全断言失败；非零，不能降级为 `BLOCKED`。
- `BLOCKED`：缺 Day-0、工具、权限、内网服务或真实 RuntimeClass/StorageClass；专用非零码并列出 `blocked: missing <decision>`。

`SKIPPED`、`NOT_APPLICABLE`、空 case 列表、命令 exit 0 但无 artifact 均不得汇总 PASS。P04 完成后公开 `poc:v1:check` 不得继续 exit 78。

### 2.3 生产完全断公网

- 只允许批准的集群内 Service DNS、内部 registry、内网 S3；模型/OIDC/对象存储/package source 不得指向互联网。
- 构建/测试环境可下载并固化工具，probe Pod 与成品 runtime 必须断网运行，不得临时 `npm install/npx/pip install/curl/git clone`。
- 附录 C D8 的 `egress-proxy` 按 DEC-02 收紧为内网审计/路由，不转发公网；公网 provider 探针必须失败。

## 3. 源码边界与产物契约

### 3.1 建议物理边界

```text
apps/runtime-orchestrator/poc/
  README.md
  poc-manifest.json
  schemas/{poc-manifest,day0,evidence-manifest,check-result}.schema.json
  config/day0.example.json
  src/
  bin/
  checks/{acquire,turn,egress,storage}/
  load/stream-turn.k6.js
  fixtures/
tests/fixtures/poc_v1/
docs/poc/20260711_V1_OpenClaw沙箱PoC证据包/
```

路径可按现有 Node workspace 微调，但不得散落临时目录、旧 `src/main` 或开发者 home。重复状态/check ID 用集中 `as const` 值对象，不用裸字符串分支。

### 3.2 PoC manifest contract

`poc-manifest.json` 是 harness 结构事实源，必须由 schema 校验并至少含：

- `schemaVersion`、`harnessVersion`、`activationTask=P04-PR4PoC脚手架`、`sourceShaPolicy=exact-head`。
- 有序且唯一的 registry：`preflight/acquire/turn/egress/storage/cleanup`；每项含 ID、argv 命令、timeout、required decisions、artifact 列表、destructive 等级。
- gVisor 默认、Kata 兼容回退的 RuntimeClass/resourceClass 映射。
- 内网 allow 类别、deny probe 类别、证据根命名规则、schema 和 retention；无成功默认端点。
- 拒绝重复 JSON key/check ID、未知高风险字段、空 registry、不存在命令/产物、危险 shell 拼接、非确定顺序、`outputs`、路径逃逸与 symlink 根目录。

### 3.3 Day-0 input contract

Day-0 区分可提交的非敏感决策和仅在受控环境注入的敏感引用。credential 只允许 Secret name/key 或环境变量**名称**，禁止明文。必需字段至少覆盖：

- 云/region、K8s minor、精确 context、专用 namespace、CNI/Pod Security、RuntimeClass、节点池约束。
- 内部 registry + immutable image digest + SBOM/签名引用、RWX StorageClass/PVC、内网 S3/MinIO bucket/prefix/KMS 声明。
- 内网 model token proxy/mock gateway、Cowork AskUser/Media/MCP bridge 回调、kube-dns；`intentionallyStubbed` 不能记 PASS。
- 公网 provider/IMDS/K8s API/DB/Redis/非批准 RFC1918/横向 Pod deny 清单。
- timeout、cleanup、retention、owner。

空值、`<待定>`、TODO、dummy/example/test.invalid、浮动 tag、无 digest、`default/kube-system` namespace、公网 URL、宽泛 CIDR 必须 BLOCKED/FAILED；example config 不能作为真实输入。

### 3.4 Probe interface contract

统一输入 `{ runId, sourceSha, startedAt, context, namespace, runtimeClass, imageDigest, evidenceRoot }`，统一 schema 输出 `{ checkId, status, startedAt, finishedAt, exitCode, assertions, artifacts, cleanupRequired, errorCode }`。

- `acquire`：render + server-side dry-run Pod/RBAC/NetworkPolicy/PVC，创建唯一 runId 资源，等待真实 gateway readiness；Pod Running/恒真 health 不算 ready。
- `turn`：只走集群内网调用真实 `chat.send { sessionKey, ... }`；至少验证 `message/messageUpdate/contextUsage/complete` 形状、顺序与唯一 complete；拒绝不存在的 `session.start/session.continue`。
- `egress`：allow/deny 成对；只允许 kube-dns 和批准内网 Service；公网、IMDS、K8s API、DB/Redis、横向/非批准 RFC1918 必须拒绝且有可观测原因；DNS rebinding 使用安全假服务。
- `storage`：为 V104 提供双向可见、20 并发小文件、单写者租约、10k 目录、事件补偿、断连重挂、路径/链接/特殊文件安全、state 隔离入口；阈值以 `08` §2.4 为准，不生成预制 PASS。
- `cleanup`：只按 runId + managed label 删除本次资源，保留证据/必要故障日志；禁止无 label namespace-wide delete。

### 3.5 Evidence bundle contract

每次执行先写同目录临时目录，schema/hash/完整性全过后原子 rename；中断不得覆盖历史成功证据。每包至少含：

- `evidence-manifest.json`：版本、source SHA、runId、时间、operator/CI run、环境分类、K8s context/namespace/CNI/Pod Security/RuntimeClass/node、image digest、SBOM/签名、StorageClass/PVC/S3 非敏感元数据。
- `results/*.json`：registry 每项各一份，有序无重复；summary 必须与子结果重算一致。
- `manifests/`：Helm render、server dry-run、实际 applied 后的脱敏 Pod/NetworkPolicy/RBAC/PVC。
- `logs/`、`metrics/`、`checksums.sha256`：每个声明 artifact 存在、非空且被哈希覆盖。
- `cleanup.json`：已删、保留、失败及重试；cleanup 失败阻断 PASS。

证据扫描必须拒绝 bearer/JWT、kubeconfig key、Docker/npm auth、PEM、Secret data/stringData、真实租户 prompt；报告不回显原文。

3. `feat(runtime): add fail-closed V1 PoC preflight`：静态 checker、工具/版本/context/namespace/RuntimeClass/StorageClass/RBAC/image 预检和三态退出。
4. `feat(runtime): add reproducible V1 probe runner`：统一 runner、四 probes、timeout/signal/process-tree/cleanup，先用 fixture executor 证明编排语义。

7. `feat(runtime): publish atomic V1 evidence bundles`：完整性、hash、脱敏、原子发布、runId 冲突与 cleanup。

## 8. 回滚、阻断与 Reviewer / Tester handoff

### 8.1 回滚

- 按 P04 切片回滚，不破坏 P03 已验收产物；chart 兼容变更回滚后重跑 P03 三门。
- 集群只删本 runId/managed-by 资源；PVC/S3 按 retention 处理，禁止删除整 bucket/namespace。
- 不能为 CI 变绿把 gate 退回 exit 78；仅协调者明确停止 P04 时才可恢复 deferred。

### 8.2 阻断/失败处理

- P03 回归失败：停 P04，返回 P03 原角色链路，不跨任务修。
- Day-0/内网/集群缺失：对应 BLK-01～BLK-06，gate 保持 BLOCKED，P04 不交 Tester/不勾总表。
- 意外公网可达、secret 入证据、cleanup 越界：立即停止，轮换可能泄漏凭据，保留脱敏失败证据并退 Developer。

### 8.3 角色交接

- Developer 按 Round 追加 Red→Green→Refactor、分 commit、命令/退出码/环境/动态证据，对 AC01～AC20 逐项定位。
