`RouteRegistry` 每项必须含 `operationId/method/path/domain/request/response/errors/auth/legacyBridgePaths/sourceRefs/support`；`ChannelRegistry` 每项含 `channelId/path/direction/messageType/schema/scope/replayable/sourceRefs`；`ErrorRegistry` 每项含 `httpStatus/retryable/userVisible/i18nKey/detailsSchema?/deprecatedSince?/replacement?`。路径参数在源码和生成物一律 `{id}`；文档 `:id` 仅在审计导入时归一。

## 5. 关键协议冻结

- 统一 REST 错误：`{error:{code,message,requestId,details?}}`；未知 code 解析失败，message 不参与分类。
- `StreamEnvelope` 固定 `type/version:1/tenantId/sessionId?/requestId?/seq?/idempotencyKey?/emittedAt/data`；`seq` 为字符串唯一补发/去重游标。
- ticket：`POST /api/v1/stream/ticket` body 仅 `sessions[]`、`resourceSubscriptions[]`；资源至少支持 `files:changed` + `{workspaceId,path?}`；response 为一次性短期 `ticket/expiresAt`。用户级事件不进 body。
- WS client control 固定 `auth/subscribe/unsubscribe/subscribeEvent/unsubscribeEvent/ping/ack`；`subscribe*` 只用 `sinceSeq`，不得出现 `cursor/lastSeq`。
- Cowork 10 项精确为 `message/messageUpdate/sessionStatus/contextUsage/goal/contextMaintenance/permission/permissionDismiss/complete/error`；每项必须同时存在 IPC topic、AsyncAPI channel、wire type、payload schema、`ElectronBridge.cowork.onStream*` 映射。`contextUsageUpdate/goalUpdate` 仅是 runtime 输入别名；`delta/tool/thinking/done/abort` 不得混入 Cowork 10 项。
- 完整 `ElectronBridge` 从现有 `IElectronAPI` 提升，保留全 property path/可选性/返回形状；preload 与未来 Web bridge 用 `satisfies ElectronBridge`。本 PR 只做类型和静态映射，不实现 Web 网络行为。
