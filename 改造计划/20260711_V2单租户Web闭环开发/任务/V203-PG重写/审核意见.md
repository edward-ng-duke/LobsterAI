# 审核意见

> 仅由独立 Reviewer 在 Developer `REVIEW_PENDING` 后按 Round 追加。Reviewer 不代写大块实现，不复用 Developer container/runId/report，不修改测试期望来消除缺陷。

## Reviewer 开始门

- [ ] V202 已由协调者 ACCEPTED；V202/P02/V107 SHA、migration/RLS/contract hashes 可重放。
- [ ] Developer 有独立有效 Red 提交；不是 V202 未过、Docker 缺失、语法/导入失败或 skip。
- [ ] Developer 实现 SHA 已冻结，工作树与证据边界明确。
- [ ] 实际 diff 未触碰 V204 服务/adapter/controller/WS、`src/main/coworkStore.ts`、`sqliteStore.ts`、V202 旧 migration、总表或其他任务记录。
- [ ] Reviewer 使用新 PG17 container、runId、fixture 和非特权 app role。

任一开始门不满足：写 `CHANGES_REQUESTED`；若唯一原因是外部环境/硬前置真实缺失，写 `BLOCKED`，不得写 `APPROVED_FOR_TEST`。

## 必审分母

| 领域 | Reviewer 独立检查 |
|---|---|
| 行为快照/compat | public surface 与 tests/callers 双向完整；golden source SHA 固定；每一差异有精确 approved policy；memory 等外部副作用明确 deferred |
| sessions | defaults/history fields、patch time、pin/list/search/keyset、batch delete、fork remap/capsule 原子性 |
| messages | createdAt+id cursor、sequence allocator、insert-before、replace、update/delete、rail、JSON/null/large payload |
| capsules/agents | CAS/revision、fork reset、logical/internal 双键、每租户 main、默认 agent 不可删、pin/delete 原子 |
| tx/concurrency | 固定锁序、whole-tx rollback、稳定 ID、bounded retry、deadlock/serialization、幂等重放 |
| tenant/RLS | FORCE+inventory+extension+复合 FK；app role 非 superuser/owner/BYPASSRLS；跨租户不泄露；池上下文不残留 |
| errors | Prisma/SQLSTATE 映射集中且脱敏；NOT_FOUND 不泄露；retryable 准确；V203 不实现 HTTP |
| shadow/cutover | 源 SQLite read-only/hash 不变；dual-write 仅 ephemeral lane；0 unexplained diff；生产 PG-only；不反灌/不冒充 V5 importer |
| N+1/perf | round-trip 上限、EXPLAIN 索引、100k fixture、p95 门、批量 fork，无 per-row query |
| 证据/范围 | PASS/FAILED/BLOCKED 语义、strict report、同 SHA、skipped=0、cleanup=true、path/import/migration allowlist |

## Reviewer 至少 12 个独立 mutation

1. V202 accepted SHA 或 migration hash 改错。
2. app role 改为 owner/BYPASSRLS，或删一表 `FORCE`/policy。
3. `set_config` 第三参改 false，复现 pool tenant leak。
4. apps deep-import generated/raw client。
5. message cursor 改为 `sequence` 或删 `id` tie-breaker。
6. append allocator 改回无锁 `MAX(sequence)+1`。
7. fork/delete/replace 在中间 fault 后 commit 半状态。
8. 改乱 agent→session→capsule→message 锁序并制造 deadlock。
9. retry 只重放最后 statement、超过 3 次或给 `P2002/P2025` retry。
10. capsule CAS 去掉 expected revision。
11. approved differences 加 wildcard/过期项或篡改 golden hash。
12. production mode 启用 SQLite/shadow/dual-write 或 PG→SQLite 反灌。
13. list/detail 引入 per-row query，或删 cursor/order 索引。
14. 替换 Testcontainers 为 mock/admin/SQLite，或把 cleanup/skipped 伪装 PASS。
15. 修改 V202 已发布 migration、V204 adapter/controller 或旧 desktop store。

至少执行 12 项，其中 2、3、5、6、7、8、9、10、12、14、15 为必选；每项应使对应测试/gate 非零，并在临时环境后恢复。

### 结论规则

- P0/P1、缺 Red、任何 tenant leak/丢写/重写/半事务、cursor 重漏、未解释 diff、性能越门、admin/mock/skip 冒充、stale evidence、cleanup 失败或范围越界：必须 `CHANGES_REQUESTED`。

- 只有 28 AC 均满足、必选 mutation 均被捕获、同 SHA 全门 Green、无 P0/P1，方可 `APPROVED_FOR_TEST`。
