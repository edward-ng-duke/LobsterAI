### 独立范围与架构审查

- [ ] diff 只含 V205 server provider/ledger、唯一 V106 renderer复用、orchestrator entrypoint/effects、必要 migration/chart/image/tests/evidence；无总体表、其它任务文档、V206 controller、新deployable、第二runtime或桌面大重写。
- [ ] desktop/server import同一 V106 renderer；若机械搬迁，前后source/golden bytes相等，没有复制、包装后行为分叉或 server-only patch。
- [ ] pure/server import graph 无 Electron、better-sqlite3、desktop stores/live getters/userData；renderer仍无fs/env/clock/random/network。
- [ ] runtime-orchestrator仍是唯一物理边界；initContainer不持K8s API token/PG admin/公网权限，不在生产安装依赖。

### PG snapshot、SecretRef 与 migration审查

- [ ] provider通过非特权 tenant transaction + RLS + `REPEATABLE READ READ ONLY`一次读取；跨租户/缺context/池复用均fail closed。
- [ ] providers/agents/IM/MCP/skills/plugins/sandbox/session/workspace/cron/callback source registry与PG权威owner双向完整；无SQLite/env/临时JSON fallback或替身领域表。
- [ ] UUID/logical `main`、session/workspace/agent关联正确；并发源修改不会产生torn snapshot，排序/hash确定。
- [ ] SecretRef覆盖provider/embedding/全部IM/MCP/plugin/callback/gateway；PG/public snapshot/ConfigMap/argv/file/log/golden/LKG无值，ref-placeholder-injection三向相等。
- [ ] migration只追加snapshot/target ledger；tenant_id/FK/RLS ENABLE+FORCE/check/unique/CAS/catalog/drift/空库-upgrade-repeat/中断/锁预算完整，无旧migration篡改或destructive down。

### Apply、并发、生命周期与LKG审查

- [ ] `/state`、`/workspace/state`、`/workspace/project` owner边界准确；AGENTS user/managed区、main/non-main、其它workspace文件effect inventory无遗漏/串写。
- [ ] allowed roots/realpath/path flavor、symlink/hardlink/编码/NUL、mode/UID/GID、只读root/PVC权限检查在任何write前完成。
- [ ] 两卷协议没有伪称全局atomic rename；prepare/fsync/hash/真实validate/ordered commit/readback/final barrier/PG finalize与journal恢复可验证。
- [ ] 任一prepare失败rename=0；partial commit完整恢复或quarantine，main entrypoint核验manifest后才启动gateway。
- [ ] leader election + workspace writer lease/fencing + PG CAS均存在；旧leader/旧Pod/旧revision/同revision异hash无法覆盖新成功。
- [ ] same revision same hash幂等、锁序/timeout/SIGTERM/cleanup安全；无无界重试、死锁或删他人temp/LKG。
- [ ] canonical diff exact；NO_OP不写/meta/revision/PID均不变；未知字段默认rebuild。
- [ ] HOT_RELOAD有真实pinned gateway PID/采纳/SLA/状态证据；provider/MCP/plugin/secret等rebuild类未静默热更；V204 drain调用稳定且未触碰V206。
- [ ] render/validate/reload/boot失败保留或恢复LKG；不兼容LKG拒绝fallback，DEGRADED不冒充PASS，invalid config无restart storm。

### Golden、真实OpenClaw、安全与证据审查

- [ ] PG→server snapshot→同一renderer与V106 golden字节一致；desktop/server shadow diff=0或exact approved pointer/file + DEC/owner/expiry，无wildcard/secret自批。
- [ ] 真实pinned OpenClaw成品镜像加载全域config并ready；非法字段mutation真实被gateway拒绝，主容器未启动，不是mock parser/schema。
- [ ] Windows pure golden与Linux container apply涵盖drive/UNC/device/Unicode/case/CRLF/POSIX；生产路径不依赖developer host。
- [ ] audit/log/metric/trace只含IDs/hash/category；secret值/前缀/正文/真实路径无泄露，encoded sentinel与image/evidence scan为0。
- [ ] 生产网络公网/metadata/K8s API/非批准内网/动态安装deny，批准内网PG/Secret/model/callback allow；无公网临时值。
- [ ] `fixture/shadow/pod apply/gateway boot/V2`证据分级明确；全部命令、exit、CST+UTC、SHA/runId/namespace/digest/report hash可复现。
- [ ] T01前置状态真实，T02～T35有有效Red→Green/mutation；AC01～AC30、coverage、lint、typecheck、全量/build、migration、Pod、rollback证据同SHA。
