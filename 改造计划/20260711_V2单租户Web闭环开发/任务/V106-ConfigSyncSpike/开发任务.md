## 2. 当前源码审计事实与切分原则

规划基线上的 `src/main/libs/openclawConfigSync.ts` 共 3502 行，当前至少混合四类职责：

1. **live 输入解析**：模块直接 import Electron `app`、provider/store/proxy/plugin 单例；`sync()` 内多次调用 Cowork/IM/agent/MCP/skills getter，可能在一次 render 内读到不同版本。
2. **渲染**：provider/model、agents、bindings、IM channels、plugins、MCP、skills、browser/tools、sandbox/session/cron，以及 AGENTS managed section 的组装。
3. **secret 处理**：`collectSecretEnvVars()` 与 `${ENV}` 引用；但现状还存在需审计的潜在明文面，如 embedding remote key、MCP env/headers、user plugin config、NIM token、网易蜂 secret、POPO aesKey 和既有 config 保留字段。不得把“legacy 如此”当安全批准。
4. **副作用与状态**：读写 `openclaw.json`、exec approvals、session stores、AGENTS/SOUL/IDENTITY/USER/MEMORY/HEARTBEAT；创建目录、删除文件、`Date.now()` 临时名、日志和 `previousBindingsJson` 内存状态。

`src/main/main.ts` 的 `getOpenClawConfigSync()` 从 Cowork SQLite、KV、IMStore、MCP cache、SkillManager 等组装 getter；`syncOpenClawConfig()` 已在外层串行并决定 secret env、hot reload/硬重启。现有 `openclawConfigSync.runtime.test.ts` 与 legacy `.mjs` 主要靠 mock Electron + 临时文件验证最终副作用，不是 Node-only 输入→输出 golden。

因此本任务遵守根 AGENTS 的超大文件政策：**禁止整文件搬运、格式化或顺手拆完 3502 行**。先建立清晰接口和 shadow parity；若某域不能在不扩大风险的情况下纯化，记录 `BLOCKED_EXTRACTION` 和精确 import/行为，不以巨大重写换绿灯。

### 3.2 Snapshot v1 必含字段

Snapshot 是一次调用内唯一事实源，先完成采集再 render；getter 不得在 renderer 中回调。至少包含：

- `schemaVersion/inputRevision` 与输入来源 hash；不得使用当前时间或随机数作为业务 revision。
- `renderContext`：注入的 `nowIso`、`pathFlavor=win32|posix`、已解析的 config/state/base/main workspace/userData/home/app/resources/cwd 路径、packaged/runtime version、代理设置、token proxy/compat proxy地址、插件/extension 可用性与预安装清单、bundle AGENTS 模板文本、skills 目录状态。
- `appState`：CoworkConfig、browser access、enterprise/session policy、raw API + 全 provider/model/server metadata、agents、skills、user plugins、MCP servers、IM settings 与所有平台实例、callback URLs/media entitlement。
- `existingState`：解析并验证后的 existing `openclaw.json`、binding fingerprint、exec approvals、各 agent session store、主/子 agent AGENTS 与 USER/HEARTBEAT 等 renderer 需要的当前内容和 hash。损坏 JSON、重复 key、非普通文件、symlink/越根在 adapter 阶段 fail closed。
- `secrets`：独立、不可 JSON 序列化到普通 fixture 的内存 secret bag；公开 snapshot 只存 `SecretRef`/env 名。所有 provider/IM/MCP/plugin/embedding/channel 凭据必须被分类，未知疑似 secret 字段不得静默进入公开 config。

采集必须 deep-copy/freeze 或等价防护；render 期间改变原 getter、数组、SQLite 行或环境变量不影响当前 snapshot。

### 3.3 纯 renderer 硬约束

纯模块及其传递 import graph 禁止：`electron`、`fs`/`fs/promises`、SQLite/store、engine manager、`process.env/cwd/platform/resourcesPath`、`os.homedir/tmpdir`、网络/child_process、`Date/new Date/Date.now`、`Math.random/randomUUID`、console 和 live getter。可使用由输入提供的纯值、确定性 hash 与无副作用 shared constants。

Renderer 返回结构化 `OpenClawConfigRenderPlan`，至少含：

- canonical `openclaw.json` 对象/UTF-8 内容、main/每 agent AGENTS.md 内容、其它明确纳入 spike 的文件 mutation；
- secret env plan（只在内存）和 config 中的 `${ENV_NAME}` 引用集合；引用与 env keys 双向对账，未引用 secret 不影响 restart 比较；
- `changedTopLevelKeys/restartImpact/bindingsFingerprint/warnings/approvedDifferenceIds`；比较时仅 `meta.lastTouchedAt` 由注入的 `nowIso` 决定，不在函数内部取时钟；
- 每个 write/delete/mkdir 的逻辑 path ID、目标相对根、expected previous hash、next hash；不直接执行副作用。

同一 snapshot 调用任意次数必须对象深等且字节相等，输入不得被 mutation。任何错误返回 typed `FAILED`，不能用部分 config。

### 3.4 effects adapter 边界

V106 adapter 只用于 shadow/temp 验证，不接管桌面生产：

- 仅允许 config root、state root 和已声明 workspace root；拒绝 `..`、绝对路径注入、Windows drive/UNC 逃逸、NUL、symlink/hardlink 非预期目标和 agent ID 路径穿越。
- apply 前按 `expectedHash` 做 CAS，防 snapshot 后外部修改；先把全部内容写到同目录唯一 temp，校验 bytes/hash，再 rename。temp ID/clock 由 adapter 注入，不进入 renderer/golden。
- 任一准备失败不得 rename；rename 阶段失败返回 `FAILED_PARTIAL_APPLY`、列出已改/未改与可执行恢复，不写 PASS。cleanup 只能删本 invocation 的 temp。
- 同 inputRevision 并发幂等；不同 revision 不允许旧 plan 覆盖新 plan。生产 queue/restart/env 注入仍留在 `main.ts`，V106 不改变。

## 4. 渲染范围与等价规则

### 4.1 必须覆盖的配置面

Golden 分母不能只取“默认 OpenAI”样例，至少双向覆盖：

- providers/models：API 形状、base URL、context/max token/custom params/cache、proxy、OAuth/API key placeholder、primary/default/agent model；生产 fixture 仅内网/保留域地址。
- agents：main 缺省与自定义、enabled non-main、identity/skills/model/cwd/workspace、稳定 main-first 排序、禁用 agent、非法/重复 ID。
- IM + bindings：Telegram/Discord/Feishu/DingTalk/QQ/WeCom/POPO/email/NIM/网易蜂/微信，multi-instance env 索引、account key 前缀碰撞、platform/instance binding、禁用/缺 credential。
- MCP：stdio/SSE/http、URL 校验、lowercase headers、CJK hash key、key collision、args/env；MCP secret 只可走 SecretRef。
- plugins/skills：preinstalled/runtime/user plugin、allow/deny/load paths、disabled managed skills、stable entries/allow、缺插件、未知 config secret。
- sandbox/session/browser/tools/cron/commands：enterprise `executionMode` 映射、heartbeat、session keepalive、proxy/SSRF、memory/dreaming/embedding、working directory与 callback。
- workspace：main/non-main AGENTS managed section、system/scheduled task/web/browser/exec/memory/heartbeat/skill creation policies；精确保留 marker 以上用户内容，marker 注入/多 marker/BOM/CRLF/空文件/模板缺失均有结论。
- 现有其它副作用（exec approvals、managed session model migration、SOUL/IDENTITY/USER/MEMORY/HEARTBEAT）必须进入 effect inventory；若不在 V106 纯渲染 Green 范围，明确标为 `legacy-adapter-owned`、V205 owner 与 golden 风险，不能无声遗漏。

### 4.2 canonical 与跨平台规则

- JSON：固定 2 空格、UTF-8 无 BOM、LF、单个末尾换行；object key 使用冻结的 code-point/ASCII comparator，不受 locale/TZ；禁止 `localeCompare` 默认 locale。
- 有集合语义的 provider/plugin/skill/MCP/channel account/binding/agent/文件 plan 必须按冻结 key 排序并拒绝重复/规范化碰撞；用户明确有顺序语义的 prompt、args、allowFrom 等保留输入顺序，但去重规则须写入 schema。
- 路径不调用宿主 `path.resolve` 猜测。snapshot 指定 `win32/posix` 并提供绝对根；覆盖 Windows drive、反斜杠、UNC、大小写策略和 Linux `/`、Unicode/空格。Golden 用逻辑 token（如 `<STATE_DIR>`）或 fixture root，禁止提交开发者 home。
- AGENTS：输出 LF + 单尾换行；最多一个精确 marker；marker 以上用户区按冻结规则保留，managed 区完全重建且幂等。Windows 输入 CRLF 与 Linux LF 在规范化后输出一致，用户正文的实质字符不得改变。

### 4.3 desktop golden 与 approved difference

- 每个 scenario 提交 `input.public.json`、运行时注入的 test-only secret factory、`expected.openclaw.json`、`expected.AGENTS.md`、`expected.plan.json` 和 source hash manifest；禁止真实 secret、token 前后缀、home、kubeconfig 或租户内容。
- 同一公开 snapshot 同时喂给 legacy oracle（受控 temp + Electron mock）和 pure renderer。比较分两门：renderer canonical bytes 必须完全确定；legacy parity 对 canonicalized JSON/AGENTS 做字段级 equality。
- 唯一默认忽略项是注入前的 `meta.lastTouchedAt` 表达差异；其它差异必须进入精确 JSON Pointer/文件/旧值类别/新值类别的 `approved-differences.json`，绑定 DEC/Reviewer/owner/到期任务。禁止 `/**` 通配、整 section 忽略或 Developer 自批。
- legacy 明文 secret 与安全 renderer placeholder 的差异不能自动批准；应 `FAILED_SECRET_LEAK` 或 `BLOCKED_SECURITY_DECISION`。fixture/diff/log 只记录字段路径、secret kind 与 hash，不记录原值。

## 9. 迁移、回滚与 V205 handoff
