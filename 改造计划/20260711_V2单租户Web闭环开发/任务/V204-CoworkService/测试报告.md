## 独立新增角度（至少覆盖 12 类）

| 用例 ID | 独立角度 | 建议故障/边界 | 预期 |
|---|---|---|---|
| `TST-01` | 首次 accepted crash window | PG commit 后、chat.send 前杀进程 | 恢复同 requestId；不重 user message/turn |
| `TST-02` | ambiguous ack | gateway 已收 send 但 client ack timeout | history/list 对账；同 key或不重发，绝不新 key双 turn |
| `TST-03` | 多 key/多 agent | 一个 session 两 key，两个 tenant 各 `main` | history/abort/delete/usage 不串 key/agent/tenant |
| `TST-04` | 多副本 permission | permission 到 A，respond 落 B，A 随后 crash | owner bus可路由/恢复；无静默；dismiss恰一次 |
| `TST-05` | AskUser race | answer 与 timeout/abort/complete 同时 | 同答幂等、异答 conflict、一个 resolve/dismiss |
| `TST-06` | 极端乱序 | chat/agent seq 各自重复、乱序、缺口、无seq | 分域重排；gap不越 checkpoint；无错误归属 |
| `TST-07` | 背压与大帧 | 高频 update+terminal、29.5/30MB边界、慢PG/Redis | 有界；terminal不丢；超限发送前拒绝 |
| `TST-08` | 四个断点 | 首帧前/中段/complete前/complete持久后断线 | message不重、complete恰一次或明确error |
| `TST-09` | owner fencing | lease过期、新旧 owner并存、旧 generation late frame | 单 owner；旧帧/命令拒绝；无双 send |
| `TST-10` | outbox replay | relay crash/restart、重复批次、Redis trim/临时不可用 | committed event最终可见；identity去重；lag告警 |
| `TST-11` | 隔离 fuzz | A/B同资源ID、body tenant篡改、cache/Redis key碰撞 | 404/拒绝，不泄露 timing/metadata，无副作用 |
| `TST-12` | secret/error | gateway nested error/URL/header/JSON string带canary | client/log/trace/report 0 secret，typed code稳定 |
| `TST-13` | 断公网 | 公网/IMDS/未批RFC1918、公共DNS、运行期下载 | 全拒；批准内网PG/Redis/gateway/OTel仍可达 |
| `TST-14` | drain/rollback | turn/permission/outbox进行中 SIGTERM/关 flag | 新请求停止；有界drain/abort/flush；恢复不双发 |
| `TST-15` | 长会话/容量 | 长 history、大 Unicode、多 tool、buffer/outbox 接近上限 | 无OOM/死锁；p95/lag/资源不越冻结阈值 |
| `TST-16` | 契约消费者 | 用 V206 风格 fake transport 调 service ports（非实现V206） | 不需业务绕行；typed result/10 events可直接消费 |

Tester 至少新增 12 类，其中必须含 `TST-02/04/06/07/08/09/10/11/12/13`；不得只复跑现有测试。测试 fixture 只用合成 nonce/脱敏数据，mutation 在隔离副本运行。
