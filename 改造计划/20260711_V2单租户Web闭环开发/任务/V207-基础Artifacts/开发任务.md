## 4. PostgreSQL metadata、归属与 blob

仅追加 V202 forward migration，不修改旧 migration。命名可服从最终 Prisma规范，语义不得缩水。

### 4.1 `artifacts`

- `id UUID`、`tenant_id/session_id/owner_user_id UUID NOT NULL`、`message_id/workspace_id UUID NULL`；`logical_key` + `generation >= 0`用于 producer幂等/流式版本。
- `type IN ('text','markdown','code','image','html')`；`status IN ('incomplete','finalizing','ready','corrupt','deleted')`；`source_kind IN ('inline','workspace_snapshot','object')`。
- 只存 canonical `rel_path`，不存宿主绝对/PVC mount/S3 key。非空 path必须 NFC、相对、无 NUL/`.`/`..`/空段/盘符/UNC/编码分隔符。
- metadata：title/file_name/language、declared/detected MIME、encoding、source size/SHA-256、ready_blob_id、finalized/deleted/created/updated。
- 复合 FK把 `(tenant,session,owner)` 绑定 session owner；message非空时绑定 `(tenant,session,message)`。若缺唯一键，只由新 migration追加。
- `(tenant,id)` 唯一；producer identity `(tenant,session,message,logical_key,generation)` partial unique；索引以 tenant开头。soft delete不立刻删 blob。

### 4.2 `artifact_blobs`

- `id/tenant_id/artifact_id`、`version >= 1`、`role IN ('source','sanitized','rasterized')`、`storage_kind IN ('inline','s3')`、`state IN ('pending','verified','committed','corrupt')`。
- exactly-one storage：inline `inline_bytes <= 65_536`；S3为服务端生成 exact object key/version；key永不出 API/log/metric。
- byte size、lowercase SHA-256、MIME、encoding、etag/checksum由服务端算；浏览器、tool metadata和 S3 ETag均不能单独作真值。
- committed blob归属/version/hash/size/MIME/locator不可更新；新内容只建新 version。ready_blob只能指同 tenant/artifact committed blob。

### 4.3 `artifact_preview_sessions`

- `id/tenant/artifact/blob/user`、contract_version、renderer_profile、created/expires/revoked/last_accessed；永远绑定一个 immutable blob version。
- credential只存 hash/nonce；即使 content GET用 V201 bearer，也逐次校验 tenant/user/artifact/blob/expiry/revoke，不能凭 ID读取。
- 到期、delete/corrupt、logout、租户切换或主动 DELETE后立即拒绝；preview session不延长 artifact生命周期。

### 4.4 RLS与写边界

- 三表 `ENABLE/FORCE RLS`，non-owner/non-superuser/non-BYPASSRLS app role；missing/invalid context fail closed。
- browser route仅 list/get/issue/content/revoke；register/finalize为 V204/internal worker port，不开放任意 create/update/delete。
- body/query/header中的 tenant/user/owner/S3 key/path override全拒；跨租户/非 owner统一404且不可时序探测。

## 5. MIME、size、hash、encoding、truncation

所有上限集中在共享 `as const` policy并由契约/服务/renderer/tests共用：

| 类别 | source上限 | preview | 拒绝 |
|---|---:|---|---|
| text/markdown/code | 25 MiB | 前2 MiB UTF-8，完整 code point/行边界；明示 truncated/returned/total bytes | binary/NUL/非法编码/错配 |
| html | 2 MiB | 完整 decode+sanitize；净化后超限仍413 | script/active/半HTML |
| raster image | 25 MiB compressed | server probe后原图或安全 derivative | decompression bomb；单边>16,384、>40MP、decoded估算>160MiB、>200 frames |
| SVG | 2 MiB | 完整 parse+sanitize，静态 image subtype | DTD/entity/script/event/foreignObject/external URL |
| inline blob | 64 KiB | 小文本/小 derivative | 拆分/塞大 BYTEA绕存储门 |

1. declared MIME、extension、magic/sniff、parser结果、type allowlist必须一致；冲突 typed unsupported/corrupt，不猜 `octet-stream`。
2. source原字节、normalized UTF-8、sanitized HTML/SVG、rasterized derivative各有独立 size/hash；S3用 checksum/version + server SHA复核。
3. 文本允许 UTF-8（含 BOM）与带 BOM UTF-16LE/BE，严格 decode后标准化 UTF-8；无 BOM非UTF-8、replacement、NUL/binary失败。
4. `truncated=true`仅 text/markdown/code。HTML/SVG/image完整或失败，不能部分解析显示。
5. hash/size与 HEAD、streamed bytes、decode任一不一致：原子标 corrupt、撤 preview、清旧 blob，不 fallback旧 hash。

## 6. 路径、symlink、incomplete与 atomic finalize

### 6.1 安全 snapshot

- 仅接受 workspace-relative candidate；拒绝 absolute、`..`、双/混 percent encoding、encoded slash/backslash、NUL、drive/UNC、空段、Unicode/case collision与保留目录。
- snapshot reader逐段 no-follow/openat2或等价；`lstat/open/fstat`同 inode、regular file、link policy/size，阻断 symlink swap TOCTOU。拒 symlink/hardlink/FIFO/socket/device/目录/root外 inode。
- preview只读 immutable blob snapshot，不读活动 PVC path；V104 PVC仍是工作文件真源，V207 blob只是 finalized版本。

### 6.2 atomic finalize

1. producer先 upsert `incomplete`；UI可显示生成中，但 preview issue返回409，content不返回半字节。
2. worker写唯一 pending object/inline row，边写边 hash/count并做 MIME/encoding/decode/sanitize/bomb检查；中断对象不可见且进 orphan cleanup。
3. S3 multipart complete后 HEAD/checksum/version，再用一个 PG transaction完成 pending→verified→committed、artifact finalizing→ready、ready_blob/outbox；DB ready不能早于 object可读。
4. 同 identity+generation+hash幂等；同 generation异hash conflict/corrupt；旧 generation迟到 complete/tombstone不能复活或退版。
5. outbox resource event含 artifact/session/state/version/hash prefix/occurredAt，不含 path/key/content；复用 V206 replay/dedupe且 Cowork 10不变。

## 8. HTML/SVG/Markdown/image安全

### 8.1 HTML iframe

- server parse/sanitize后生成独立 sanitized blob；移除 script、frame/iframe、object/embed、form/control、base、meta refresh、link、style import、event、javascript/vbscript/file/blob/http(s) URL、SVG active node、CSS `url/@import/expression/behavior`。
- iframe固定 `sandbox=""`，不含 scripts/same-origin/top-navigation/popups/forms/downloads；不注入 hash-navigation script，不给 parent/opener/storage/cookie/clipboard/service-worker能力。
- 文档注入并测试 CSP：`default-src 'none'; script-src 'none'; connect-src 'none'; img-src data:; font-src data:; media-src 'none'; object-src 'none'; frame-src 'none'; child-src 'none'; worker-src 'none'; manifest-src 'none'; base-uri 'none'; form-action 'none'; style-src 'unsafe-inline'; sandbox`；另设 nosniff、严格 frame-ancestors/CORP。
- Chromium/WebKit/Firefox必须证明 script/eval/worker、fetch/XHR/WS/remote asset/DNS、meta/base/form/open/download/top-nav均0副作用。

### 8.2 其它 renderer

- SVG禁 DTD/entity并重序列化；href只允许同文档 fragment或受限 image data URI；禁 external refs/CSS URL/foreignObject/animation/event。无安全 renderer时明确unsupported，不 raw innerHTML。
- Markdown禁 raw HTML或走同 sanitizer；link不自动请求；image仅同租户已注册 artifact/data-safe derivative；禁 file/localfile/javascript/data:text/html/blob、remote embed、Mermaid执行。
- code/text只用 text node/readonly editor；高亮包离线预装，不运行/lint/网络加载。
- raster在浏览器 decode前 server probe；异常图生成受限静态 derivative或拒绝。decode失败清旧图，revoke object URL并显示 typed error。

## 12. migration、rollout、rollback、cleanup

1. preflight锁 V202 schema/RLS、contract、PG17、S3 tuple；drift/failed migration/缺能力即阻断。
2. 只追加 artifacts/blobs/preview sessions、FK/index/check/RLS；empty、V202→V207、repeat、旧V204/V206 app兼容均通过；不得db push/改旧migration/RLS未FORCE接流量。
3. flags默认关：schema→internal finalize shadow→text/code→markdown/image→HTML；每类kill switch独立，安全policy不可被flag放宽。sanitizer policy version进入blob/cache/report。
4. rollback先停issue、撤active preview、关type/总flag、drain pending。保留forward schema/metadata/ready blob/outbox，不down/drop、不回灌SQLite、不回raw/旧sanitizer/公网CDN/allow-scripts。
5. sanitizer/decoder漏洞立即禁type、revoke、隔离derivative并forward-fix；恢复同identity/hash幂等，旧generation不得复活。
6. cleanup只删本run pending preview/blob/temp/prefix，按exact ID/version/label幂等；不得通配共享对象。cleanup非零=FAILED。

## 13. 四角色状态机

- Planner仅冻结本目录四文件；不写实现或PASS。
- Developer仅在硬门有效后按统一Round追加 Red→Green→Refactor、mutation、命令/exit/hash、migration/rollback/cleanup与逐条回复。
- Reviewer使用独立 Agent/checkout/PG schema/S3 prefix/browser profile/runId做独立mutation；P0/P1、缺Red、tenant leak、active content、script/network/top-nav、stale cache、hash错、mock/public S3、skip、越界类型或cleanup失败均不能批准。
